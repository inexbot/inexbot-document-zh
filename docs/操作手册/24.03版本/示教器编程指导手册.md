# 示教器编程指导教程

## 1 开机安全确认

1.  检查伺服、控制器、示教盒各部件连接线是否已连接完好。

2.  把机柜面板上的主电源开关旋转到接通（ON）的位置，主电源接通。

3.  按下机柜面板上的绿色伺服启动按钮。

4.  示教器开机成功后确认机器人参数是否正常，参数没有问题后就可以点动机器人。

| 警告：                                                                     |
| :--- |
| ![](./assets/jxxdsk3lu8u0yljvqehy7.png) |
| 防止机器人在运动过程中出现飞车现象，示教前请确认急停按钮正常！             |

## 2 急停按钮的使用确认

1.  在机器人使用前，请分别对控制柜、示教盒上的急停按键进行确认，按下时，伺服电源是否断开。

2.  按控制柜及示教盒上的急停按键。

3.  确认伺服电源关闭，示教器显示伺服报错，控制柜伺服报错灯亮。

4.  清除伺服错误，控制柜伺服报错灯灭，示教器上显示"伺服停止"。

5.  轻按示教器的【DEADMAN】按键（示教器背后的按键），机器人上电，示教器上显示"伺服运行"，表示伺服电源成功接通。

## 3 机器人配置参数

1.  示教器开机成功后，在【设置】-【机器人参数】-【从站配置】界面确认连接的从站个数、从站型号、机器人类型。如图所示：

![](./assets/d5csdgg77z8lzfjnondro.png)

![](./assets/rpcsz5ljlz2_mekbijzgs.png)

2.  在【设置】-【机器人参数】-【DH参数】界面确认杆长参数、耦合比。

![](./assets/vaqozjfjqqc36ugyjjugs.png)

3.  在【设置】-【机器人参数】-【关节参数】界面确认关节参数。

![](./assets/49kwxfdbfqzg2mhvoby8d.png)

4.  示教器开机成功后如果机器人配置参数没有的话，就需要重新导入配置。

导入配置流程：

1.  点击【设置】-【机器人参数】-【从站配置】-【机器人】选择机器人类型

2.  点击【设置】-【系统配置】-【导入配置】选择配置文件，系统重启后机器人参数配置导入成功

| 警告：                                                                                     |
| :--- |
| ![](./assets/q1hr1dfoir692vtj3pyoj.png) |
| 导入的配置文件必须是当前选中机器人的配置文件，配置文件不对会导致在点动机器人时出现飞车现象 |

配置文件导入成功后的注意事项：

1.  伺服顺序

进入【设置】-【机器人参数】-【从站配置】-【机器人设置】。

点动机器人，查看J1是否控制的一轴，J2是否控制的二轴，以此类推，若不是自行修改。

注意：某些伺服从站存在多合一，从站配置里面机器人轴未必是1234567按照顺序。

2.  模型方向：

当点动机器人的实际方向与机器人示例图片方向一致时，模型方向数值不变。

当点动机器人的实际方向与机器人示例图片方向相反，需要在关节参数界面将模型方向数值取反。

3.  零点位置：

点击示教器上面的【零点】，机器人运行到零点位置，机器人回到零点位置后看一下机器人每个轴的位置是否都在零点，如果零点位置有偏差需要重新校准零点。

| **注释：**                                                                                   |
| :--- |
| ![](./assets/tbykr5epgjup9ionrtvi0.png) |
| - 若出现实际点动机器人关节轴90度，但示教盒上显示的并不是90度，则需要调整减速比或与厂家确认。 |
| - 若出现在直角坐标系下，点动坐标轴，机器人走不直的情况，则需要调整DH参数或联系厂家确认。     |

## 4 点动机器人操作

1.  示教器与控制器连接正常。

2.  伺服、机器人参数正常。

3.  在示教模式下，按下示教器上的【伺服】按键，将伺服停止状态切换至伺服就绪。

4.  轻按示教器的【DEADMAN】按键（示教器背后的按键），不松手，此时听到机器人上电的声音，且"伺服状态"一栏显示为绿色的"伺服运行"。

5.  通过操作示教器右侧物理按键来控制机器人运动。

## 5 程序指令界面修改

### 5.1 工程界面操作

1.  用户权限切换至管理员或者新建用户自定义权限。用户如何自定义权限可以在《功能键+坐标系》这一节查看。

2.  点击左侧的【工程】，进入工程预览界面。

![](./assets/l-jcznf6uk2rdhvat40y4.png)

#### 5.1.1 新建

1.  点击【工程】进入工程界面。

2.  点击新建，进入新建程序界面。

3.  输入程序名称后点击底部的【确定】按键，程序创建成功，并跳转到程序指令界面；若想要取消新建程序，则点击【取消】按键。

注意事项：程序名称必须为以字母/汉字开头的两位及以上的字符串，新建程序名称不能为已有程序的名称，程序名不能超过30位。

程序创建成功后在工程界面就会出现新建的程序名称。

![](./assets/-pqnim6gbsegd86zhtcrx.png)

![](./assets/u7mqiryjvsqxdydceilhg.png)

#### 5.1.2 打开

1.  点击【工程】，进入工程预览界面。

2.  选中想要打开的程序。

3.  点击底部的【打开】按键，程序打开成功。

![](./assets/8cucqsntmyrewrpwyxwgi.png)

![](./assets/ii_9gi0gomia9lfsteqjq.png)

#### 5.1.3 删除

1.  点击【工程】，进入工程预览界面。

2.  选中需要删除的程序。

3.  点击底部的【删除】按键，点击提示界面【确定】程序被删除，点击【取消】不会对选中的程序执行删除操作。

![](./assets/aeyp7duhi47jdcmpydssg.png)

![](./assets/ghaw2rzexlz1h-bfajqt8.png)

#### 5.1.4 操作

点击操作按钮可以对程序执行复制、重命名、批量删除、设为主程序操作。

- 复制:用户复制已有的程序，支持用户在修改程序之前对原有的程序做一个备份操作。

1.  进入【工程】界面。

2.  选中需要复制的程序.

3.  点击底部的【操作】按键，选择【复制】，在复制程序提示界面点击【确定】程序复制成功，点击【取消】则取消程序的复制操作。复制的程序名称用户也可以修改。

4.  复制成功后在工程界面会显示复制的程序。

![](./assets/lgrtdi5pudl_y7zboieh8.png)

![](./assets/zrvzrz57ubiibhsvz1uda.png)

- 重命名：对选中的原有程序进行重命名操作

注意事项：重命名的程序的程序名不能为已有程序的名称，前后台的程序的程序名不能重复.

1.  点击【工程】，进入工程界面

2.  选中需要重命名的程序

3.  点击底部的【操作】按键，选择【重命名】，在工程预览界面重新给程序命名

4.  点击程序预览界面的【确定】，程序重命名成功。原有的"程序9"重命名为"测试程序9"，点击【取消】则取消程序的重命名操作

![](./assets/vdzhdpwl0cxazyhdtcmg8.png)

![](./assets/vpwdssvakvs8ii9bblg7r.png)

3.  批量删除：一次可以删除多个程序，批量选中仅能选中当前页的程序，无法删除上一页或下一页的程序。

- 点击【工程】，进入工程界面。

- 点击底部的【操作】按键，选择【批量删除】。

【全选】：选中当前页面的所有程序，删除当前页面的部分程序可以直接点击当前页面的程序或者示教器右侧的滚轮键选择程序。

【反选】：如下图，如果现在选中的程序是程序9、程序8、程序7、程序6、程序5、程序4，点击反选会选中程序3、程序2、程序11、程序10。

【取消】：取消对程序的批量删除操作。

【确定】：点击确定按键后在弹出的提示框中点击【确定】按键则选中的程序批量删除成功，点击【取消】退出批量删除。

![](./assets/u24x0b8fp6vh8mqkngtkl.png)

![](./assets/xteujjm3nnv__oepiv66e.png)

4.  设为主程序。

如果在程序里面插入了较多的调用子程序指令，设为主程序这个操作方便用户在运行程序时知道哪一个程序是主程序。

- 点击【工程】，进入工程界面，选中程序。

- 点击底部的【操作】按键，选择【设为主程序】，设为主程序的程序名前会带一个"\*"号。

![](./assets/up2e2e2qegqglwr6nyq-v.png)

## 5.2 程序指令界面操作

点击【工程】，进入工程预览界面。

选中建立的程序名称，点击【打开】，进入程序指令界面。

![](./assets/aidkgmhb_igevp1gpbwze.png)

### 5.2.1 插入指令

1.  在程序指令界面点击【插入】，进入指令插入界面。

2.  选择需要插入的指令类型，然后选择指令。

例如：运动控制类-直线指令。

3.  点击【确定】，进入指令参数设定界面。

4.  在参数设定界面点击【确定】指令插入成功。

5.  插入其它类型的指令步骤都是一样的。

![](./assets/i5aqfhwmmruy7iqq8qrz7.png)

### 5.2.2 修改指令

1.  程序指令界面选中需要修改的指令，点击【修改】，进入参数设定界面。

例如：需要将选中的直线指令速度修改为100mm/s，可以在参数设定界面速度行修改指令速度。

2.  修改完参数后，点击【确定】指令参数修改成功。

![](./assets/dyejkf6ugeikxpm2g8brd.png)

1.  程序指令界面选中需要删除的指令，点击【删除】。

### 5.2.4 操作

1.  【复制】：

- 程序指令界面选中需要复制的指令，点击【操作】，选中【复制】。

- 点击【粘贴】，将复制的指令粘贴到目标位置。

![](./assets/9rpz8a6avnoup4dwqplwa.png)

2.  【粘贴】：将复制或者剪切的指令粘贴到目标位置。

3.  【剪切】：

- 选中需要剪切的指令，点击【操作】，选中【剪切】。

- 点击【粘贴】，将复制的指令粘贴到目标位置。

- 如下图将第3行指令剪切后粘贴到当前作业文件的第一行。

5.  【下移】：点击下移将当前选中的指令向下移动，点击一次指令向下移动一行。

6.  【注销】：

- 选中需要注销的指令，点击【操作】，选中【注销】。

- 点击【注销】选中的指令被注销，注销后的指令在运行时会跳过，不会执行。

- 点击【从此运行】，光标在选中的指令序号前。

## 7 单步运行

1.  点击【工程】，打开运行的程序，进入程序指令界面.

2.  选中要进行单步操作的指令行。

3.  按下【DEADMAN】按键，机器人上电。

4.  按下【单步】按键，机器人执行选中行的指令，执行完后停止。

5.  选中行自动下移/上移，若要单步运行下一行指令则再按一次【单步】 按键。

正序运行：

如下图所示，点击【单步】程序从第1行开始运行，第1行运行结束后自动跳行到第2行，再点击【单步】运行第2行指令，依次运行完整个作业文件。

![](./assets/nhxyzptp5oxxh3sqstdko.png)

倒序运行：

如下图所示，点击【单步】程序从第4行开始运行，第4行运行结束后自动跳行到第3行，再点击【单步】运行第3行指令，第一行指令运行结束后整个作业文件执行结束。

![](./assets/be0e3jlvo1bb1xxjqiqv1.png)

## 8 试运行

1.  试运行功能是示教模式下将【启动】键作为试运行按键，上电按住【启动】键保持运行，松开停止。

2.  试运行模式支持所有指令。

3.  试运行不支持倒序以及后台程序。

如下图所示，一直按下【DEADMAN】按键和【启动】按键会运行完作业文件里面的所用指令。

### 10.1.1 加速度参数

加速度倍数越大，表示机器人到达最大加速度的时间越快。

此处的全局速度表示示教器上方状态栏的速度。

![](./assets/udw9yt-nnqi8io8xiu5pz.png)

1.  点到点指令修改加速度参数

进入【设置-机器人参数-关节参数】调整最大加速度参数

![](./assets/ni9x1s51bssedtqh8izpr.png)

- 最大轴速度=额定速度\*全局速度\*指令速度。

例如：点到点指令，全局速度为50%、指令速度为40%、指令加速度为10%、额定速度为200°/s、最大加速度为4倍。

最大轴速度=200°/s\*50%\*40%=40°/s。

运行到额定速度时间=（额定速度\*全局速度\*指令速度）/（额定速度\*最大加速度\*全局速度\*指令加速度）=（200°/s\*50%\*40%）/（200°/s\*4\*50%\*10%）=1s。

3.  直线指令修改最大加速度参数。

进入【设置-机器人参数-笛卡尔参数】调整最大加速度参数。

![](./assets/dsi_s6ueqmzbg2te6k6-k.png)

- 最大线速度=全局速度\*指令速度。

例如：直线指令，全局速度为30%、指令速度为1000mm/s、指令加速度为50%、笛卡尔最大速度为2000mm/s、笛卡尔最大加速度为2倍。

最大线速度=全局速度\*指令速度=30%\*1000mm/s =300mm/s。

运行到额定速度时间=（全局速度\*指令速度）/(笛卡尔最大速度\*笛卡尔最大加速度\*指令加速度\*全局速度)
=（30%\*1000mm/s）/（2000mm/s\*2\*50%\*30%）=0.5s。

4.  根据最大正转速，减速比计算额定正速度。

额定正速度=额定正转速 \* 360 / 60 / 减速比。

## 12 断点运行

断点运行：运行程序时进行下电或者其他导致程序运行中断的操作，当程序再次启动运行时从运行中断的点继续运行。

运行模式断点运行举例说明：

![](./assets/-qbf3sltih72ftilyzwze.png)

1.  运行模式程序运行过程中（第一条指令除外）,切换至其他模式时导致运行中断，会将中断时的变量状态、程序运行位置存为断点，断点执行不会清状态。

2.  再次切到运行模式点击【启动】时会弹出提示框，选择"断点执行"则从断点处继续运行，选择"重新运行"则断点消失从第一条指令重新运行。

3.  清断点操作：

- 运行模式进行回零、复位、运行其他程序、运行到该点、重启控制器、修改机器人参数。

- 运行模式切到示教模式进行指令的插入/删除/移动/剪切/复制粘贴操作。

- 运行模式切到示教模式数值变量/位置变量/程序指令参数。

- 控制器报错。

例如：程序在运行中切模式，进行回零操作，然后再次切运行发现清除断点，如下图。

![](./assets/wakxhglfahmshy-gkirmk.png)

4.  不清除断点操作

例如：

- 运行模式下IO紧急停止/伺服警报/输出信息指令。

- 程序运行时下电退出当前作业文件程序，然后重新运行退出的作业文件。

- 程序运行时下电，切到示教模式点动机器人，然后重新运行退出的作业文件等操作。

### 13.2.4 Modbus与io优先级

![](./assets/ovz5b-wey-ot2oxdyk7nx.png)

当modbus和io都连接的话，两者远程启动程序时是有优先级的。

如何修改优先级？

导出配置，在config文件夹里面打开modbusAddr.json文件，找到如下参数：

![](./assets/rq1beoknaxdd20la1r6fq.png)

![](./assets/erpgeckohnwk5wlponlua.png)

- coexistIOControl：false

表示modbus与IO不共用，modbus优先级高，即当modbus连接上时IO不能控制机器人。

- coexistIOControl：true

表示modbus与IO共用，即modbus和IO可以同时控制机器人。

当modbus与IO共用时，此时调取作业文件调取的是【远程程序设置】中的作业文件，同时【操作参数中-预约模式】必须打开。

- coexistIOControl：false

- modbusPriorityHigh：false/true

modbus默认优先级高,且modbus连接上时IO不能控制。

- coexistIOControl：true

- modbusPriorityHigh：false

表示
modbus和IO共用且IO优先级高，即modbus和IO可以同时控制机器人且modbus按照IO的设置（示教盒上的设置）运行，例如断点与当前行运行。

- coexistIOControl：true

- modbusPriorityHigh：true

表示IO不能运行自己的程序，而是io(在示教器上的设置）控制modbus选择的程序以及运行次数和总次数。
且在共用时，只要优先级高的那一方未连接，另一方功能不可用。

## 常见问题解答

### Q1: 如何切换示教器的操作模式？

**A1:** 示教器的操作模式通过右上方的旋钮进行切换：
- 左侧：示教模式，用于编程和调试
- 中间：运行模式，用于执行程序
- 右侧：远程模式，用于外部控制

### Q2: 如何在示教模式下点动机器人？

**A2:** 在示教模式下，按住DEADMAN按键，然后使用示教器上的方向键或摇杆进行点动操作。可以在点动速度设置中调整点动速度。

### Q3: 如何创建新的作业文件？

**A3:** 在示教模式下，点击【工程】-【新建】，输入文件名后点击【确定】即可创建新的作业文件。

### Q4: 如何插入指令到作业文件中？

**A4:** 在示教模式下，打开作业文件，点击【指令】按钮，选择需要的指令类型，设置参数后点击【确定】即可插入指令。

### Q5: 如何设置程序的运行次数？

**A5:** 在运行模式下，点击【设置次数】，输入需要的运行次数后点击【确定】。设置为0表示循环运行。

### Q6: 如何使用单步运行功能？

**A6:** 在示教模式下，打开作业文件，选中要执行的指令行，按住DEADMAN按键，然后点击【单步】按钮，机器人会执行选中的指令。

### Q7: 如何使用断点运行功能？

**A7:** 当程序运行中断后，再次启动时会弹出提示框，选择"断点执行"则从断点处继续运行，选择"重新运行"则从第一条指令重新运行。

### Q8: 如何通过远程模式启动程序？

**A8:** 可以通过IO信号或Modbus协议启动远程模式下的程序：
- IO启动：设置远程IO功能，通过外部IO信号触发程序运行
- Modbus启动：通过Modbus协议写入对应地址码启动程序

### Q9: 如何调整示教器的速度？

**A9:** 可以通过示教器状态栏上的速度滑块调整全局速度，也可以在设置-操作参数中修改默认速度。

### Q10: 如何处理程序运行中的报错？

**A10:** 当程序运行中出现报错时，可以通过以下方式处理：
- 查看报错信息，了解具体原因
- 清除报警：点击【清除报警】按钮或通过远程IO信号清除
- 检查程序逻辑和参数设置，修复问题后重新运行
