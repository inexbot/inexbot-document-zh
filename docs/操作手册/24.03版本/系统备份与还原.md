# 系统自动备份与还原

**打开设置——系统设置——自动备份界面，可以备份系统所需要的文件。**

**控制器自动备份功能：**
- 备份内容：程序、参数。
- 备份个数：最大10个。
- 备份命名：按前提、版本、时间命名。如2024年12月9日10点36分修改参数，备份名“参数-22.08-5.8.0-20241209103650
- 触发备份的前提：开机、修改参数、修改程序、升级。

**备份频率：**
- 开机时确认版本、参数正常后备份一次；
- 修改参数后5分钟内没有再次修改参数备份一次；
- 修改程序（插指令、修改指令）后5分钟内没有再次修改备份一次；升级前备份一次。

**还原选中备份：** 选中备份的参数，点击还原选中备份，可以还原选中的备份参数配置。

**U盘备份：** 将U盘插入示教器USB口，点击U盘备份，根据备份时当前运行的系统程序定义SystemBackup的文件名前缀为'T30-2-SystemBackup04-10-13-08'

**U盘还原：** 将U盘插入示教器USB口，点击U盘备份，选择需要还原的备份文件，上传后相关参数还原成备份文件中的参数

**备份的文件：**
- 示教器备份文件：config.db；Pinyin.db；ServoError.db；TColor.db；Userdata.db；T31_ARM_Linux_Teach-pendant（示教器程序）。

- 控制器备份文件：控制器配置文件config；工艺文件craft；eni文件；作业文件job；lua文件；多语言文件msg_languages；预置参数preset；变量variant；控制器程序C1102_X86_Linux-RT_Robot-controller；userDefinedProfile.json文件

**备份流程：**

将备份文件全部导出，存放在T30-1/T30-2-SystemBackup中，根据备份系统当前运行的系统程序，定义SystemBackup的文件名前缀为T30-1还是T30-2。nrc对应T30-1-SystemBackup，nrc2对应T30-2-SystemBackup

**备份路径：** U盘根目录

**还原流程：**

先获取系统当前运行的程序信息，对比备份系统文件的文件名。即nrc对应T30-1，nrc2对应T30-2。当前运行的系统与备份系统文件名匹配时，则允许系统还原，反之则不允许系统还原。

恢复的逻辑为先删除原先的文件，再上传备份的文件。

**Q:控制器自动备份的内容包括哪些？**

A:程序和参数。

**Q:U盘还原后导致控制器崩溃**

A:还原前确认备份的参数与当前版本是否对应、硬件设备是否一样、机器人型号是否一样。
