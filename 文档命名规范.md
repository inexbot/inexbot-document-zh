# 文档命名规范

## 1. 命名原则

- **清晰明了**：文件名应准确反映文档内容，便于快速识别。
- **一致性**：所有文档应遵循统一的命名格式。
- **无特殊字符**：避免使用空格、特殊符号等，使用下划线或连字符分隔单词。
- **版本控制**：对于有多个版本的文档，应在文件名中包含版本信息。

## 2. 命名格式

### 2.1 一般文档命名格式

```
[类型]-[主题]-[版本].md
```

**示例**：
- `产品-工业机器人控制器C2200系列-1.0.md`
- `技术-通讯协议与地址码-1.0.md`
- `操作-六轴协作机器人操作手册-22.07.md`

### 2.2 版本号格式

- **主版本号**：重大更新，如架构变更、功能大幅增加。
- **次版本号**：小功能更新、bug修复。
- **修订号**：文档内容修正、格式调整。

格式：`X.Y.Z`

**示例**：
- `1.0.0`：初始版本
- `1.1.0`：增加了新功能
- `1.1.1`：修正了文档错误

### 2.3 日期格式

对于时间敏感的文档，可在文件名中包含日期：

```
[类型]-[主题]-[YYYYMMDD].md
```

**示例**：
- `操作-2207人机协作手册-20260318.md`

## 3. 命名规范

### 3.1 类型标识

- **产品**：产品资料相关文档
- **技术**：技术资料相关文档
- **操作**：操作手册相关文档
- **解决方案**：行业解决方案相关文档
- **指南**：使用指南、最佳实践相关文档

### 3.2 主题命名

- 使用简洁明了的中文描述
- 避免使用过长的文件名
- 突出文档的核心内容

### 3.3 特殊情况

- **目录命名**：使用中文描述，避免使用数字或特殊字符
- **图片文件**：使用描述性名称，如 `C2200产品图片.png` 而非 `image1.png`
- **压缩文件**：与对应文档保持相同的命名格式，后缀改为 `.zip`

## 4. 示例

| 文档类型 | 推荐命名 | 不推荐命名 |
| :--- | :--- | :--- |
| 产品资料 | `产品-工业机器人控制器C2200系列-1.0.md` | `C2200.md`、`控制器文档.md` |
| 技术资料 | `技术-通讯协议与地址码-1.0.md` | `通讯协议.md`、`协议文档.md` |
| 操作手册 | `操作-六轴协作机器人操作手册-22.07.md` | `操作手册.md`、`六轴机器人.md` |
| 解决方案 | `解决方案-焊接自动化解决方案-1.0.md` | `焊接方案.md`、`解决方案.md` |
